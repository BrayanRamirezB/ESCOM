Diagramas de clase

Definición de diagrama. La clase se representa por un rectángulo, que se divide en tres rectángulos. El nombre de la clase se da en el rectángulo superior. los atributos se muestran con sus nombres y sus tipos en el segundo cuadro. La tercera caja muestra los métodos con sus tipos de retorno y parámetros (nombres y tipos). El acceso especificador para cada campo y método se proporciona justo delante del nombre del campo o del método. El signo - indica acceso privado, el signo + significa acceso público y el signo # se utiliza para el acceso protegido.

La clase Television.


       public class Television {
         int canal = 1;
         int nivelVolumen = 1;
         boolean on = false;

         public Television() {
         }
         public void ponerOn() {
           on = true;
         }
         public void ponerOff() {
           on = false;
         }
         public void setCanal(int nuevoCanal) {
           if(on && nuevoCanal >= 1 && nuevoCanal <= 120)
             canal = nuevoCanal;
         }
         public void setVolumen(int nuevoNivelVolumen) {
           if(on && nuevoNivelVolumen >= 1 && nuevoNivelVolumen <= 7)
             nivelVolumen = nuevoNivelVolumen;
         }
         public void siguienteCanal() {
           if(on && canal < 120)
             canal++;
         }
         public void anteriorCanal() {
           if(on && canal > 1)
             canal--;
         }
         public void masVolumen() {
           if(on && nivelVolumen < 7)
             nivelVolumen++;
         }
         public void menosVolumen() {
           if(on && nivelVolumen > 1)
             nivelVolumen--;
         }
       }

Herencia


Definición de herencia. La programación orientada a objetos le permite derivar nuevas clases a partir de clases existentes. Esto es llamado herencia. La herencia es una característica importante y poderosa en Java para reutilizar software. Suponga que debe definir clases para modelar círculos, rectángulos y triángulos. Estas clases tienen muchas características comunes. ¿Cuál es la mejor manera de diseñar estas clases para evitar redundancia y hacer que el sistema sea fácil de comprender y de mantener? La respuesta es utilizar la herencia.


Superclase y subclase. Utiliza una clase para modelar objetos del mismo tipo. Las diferentes clases pueden tener algunos propiedades y comportamientos, que pueden generalizarse en una clase que pueden ser compartidos por otras clases. La herencia le permite definir una clase general y luego extenderla a clases más especializadas. Las clases especializadas heredan las propiedades y métodos de la clase general. Considere los objetos geométricos. Suponga que desea diseñar las clases para modelar formas geométricas. Objetos como círculos y rectángulos. Los objetos geométricos tienen muchas propiedades comunes y comportamientos. Se pueden dibujar en un color determinado, rellenos o sin rellenar. Por lo tanto, se puede usar una clase general FiguraGeometrica para modelar todos los objetos geométricos. Esta clase contiene las propiedades color y relleno y sus métodos get y set apropiados. Suponga que esta clase también contiene la propiedad datoCreado() y los métodos getDatoCreado() y toString(). El método toString() devuelve una representación de cadena para el objeto. Dado que un círculo es un tipo especial de objeto geométrico, comparte propiedades y métodos comunes con otros objetos geométricos. Por tanto, tiene sentido definir la clase Circulo que amplía la clase FiguraGeometrica. Asimismo, Rectangulo también se puede declarar como una subclase de FiguraGeometrica. El diagrama de clases muestra la relación entre estas clases. Se usa una flecha que apunta a la superclase para denotar la relación de herencia entre las dos clases involucradas. En terminología de Java, una clase C1 extendida de otra clase C2 se llama subclase y C2 se llama superclase. Una superclase también se conoce como clase padre o clase base, y subclase como una clase secundaria, una clase extendida o una clase derivada. Una subclase hereda campos de datos y métodos de su superclase y también pueden agregar nuevos campos de datos y métodos. La clase Circulo hereda todos los campos y métodos de datos accesibles de la clase FormaGeometrica. Además, tiene un nuevo campo de datos, radio y los métodos get y set asociados. También contiene los métodos getArea(), getPerimetro() y getDiametro() para devolver el área, perímetro y diámetro del círculo. La clase Rectangulo hereda todos los campos y métodos de datos accesibles de la Clase FiguraGeometrica. Además, tiene los campos de datos ancho y alto y los métodos get y set asociados. También contiene los métodos getArea() y getPerimetro() para devolver el área y el perímetro del rectángulo. Las clases FiguraGeometrica, Circulo y Rectangulo, estan descritas por su código y presentadas en el diagrama de clases.


La clase FiguraGeometrica.


       public class FiguraGeometrica {
         private String color = "blanco";
         private boolean rellenado;
         private java.util.Date datoCreado;
         /** Constructor */
         public FiguraGeometrica() {
           datoCreado = new java.util.Date();
         }
        /** Constructor con un color especifico * y valor rellenado */
         public FiguraGeometrica(String Color, boolean rellenado) {
           datoCreado = new java.util.Date();
           this.color = color;
           this.rellenado = rellenado;
         }
         /** Color retornado */
         public String getColor() {
           return color;
         }
         /** Poner un nuevo color */
         public void setColor(String color) {
           this.color = color;
         }
         /** Rellenado regresado. De un rellenado boolean,
           su metodo get es nombrado esRellenado */
         public boolean esRellenado() {
           return rellenado;
         }
         /** Poner un nuevo rellenado */
         public void setRellenado(boolean rellenado) {
           this.rellenado = rellenado;
         }
         /** Conseguir datoCreado */
         public java.util.Date getDatoCreado() {
           return datoCreado;
         }
         /** Regresa una cadena que representa a este objeto */
         public String toString() {
           return "creatdo en " + datoCreado + "\ncolor: " + color +
             " y rellenado: " + rellenado;
         }
       }

La clase Rectangulo.


        public class Rectangulo extends FiguraGeometrica {
          private double ancho;
          private double alto;
          
          public Rectangulo() {
            
          }

          public Rectangulo(double ancho, double alto) {
            this.ancho = ancho;
            this.alto = alto;
          }

          public Rectangulo(double ancho, double alto, String color, boolean rellenado) {
            this.ancho = ancho;
            this.alto = alto;
            setColor(color);
            setRellenado(rellenado);
          }
          
          public double getAncho() {
            return ancho;
          }

          public void setAncho(double ancho) {
            this.ancho = ancho;
          }

          public double getAlto() {
            return alto;
          }

          public setAlto(double alto) {
            this.alto = alto;
          }

          public double getArea() {
            return ancho * alto;
          }

          public double getPerimetro() {
            return 2 * (ancho + alto);
          }

        }

Otro ejemplo de herencia:

        class A {
          int i, j;
          
          void mostrarij() {
            System.out.println(" i y j: " + i + " " + j);
          }
        }
        
        class B extends A {
          ink k;
          
          mostrark() {
            System.out.println(" k: " + k);
          }
          
          void suma() {
            System.out.println("i + j + k : " + (i + j + k));
          }
        }
        
        class HerenciaDriver {
          public static void main(String args[]) {
            A superO = new A();
            B subO = new B();
            
            superO.i = 10;
            superO.j = 20;
            System.out.println(" Contenido de superO: ");
            superO.mostrarij();
            System.out.println();
            
            subO.i = 7;
            subO.j = 8;
            subO.k = 9;
            System.out.println(" Contenido de subO: ");
            subO.mostrarij();
            subO.mostrark();
            System.out.println();
            
            System.out.println(" La suma de i, j y k en sub=: ");
            subO.suma();
          }
        }

La salida es:

Contenido de superO:
i y j : 10 20
Contenido de subO:
i y j: 7 8
suma de i, j y k en subO:
i+j+k: 24
