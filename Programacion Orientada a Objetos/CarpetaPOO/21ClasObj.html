La clase

Definición de clase de la Real Academia Española. 1. f. Conjunto de elementos con caracteres comunes, 8. f. Distinción, categoría.


Definición de clase en computación. Es la agrupación de objetos. Las clases permiten que un diseñador de software considere los objetos como diferentes tipos de entidades. Visita objetos de esta manera nos permite utilizar los mecanismos de clasificación para categorizar estos tipos, definir jerarquías y comprometerse con las ideas de especialización y generalización de objetos.


Object-Oriented Analysis and Design, Sarnath Ramnath and Brahma Dathan, Pág. 6


La programación orientada a objetos (POO) implica la programación mediante objetos. Un objeto representa una entidad en el mundo real que se puede identificar claramente. Por ejemplo, un estudiante, un un escritorio, un círculo, un botón e incluso un préstamo pueden verse como objetos. Un objeto tiene una única identidad, estado y comportamiento. class.


El estado de un objeto (también conocido como propiedades o atributos) está representado por campos de datos con sus valores actuales. Un objeto circulo, por ejemplo, tiene un campo de datos radio, que es la propiedad que caracteriza a un círculo. Un objeto rectángulo tiene campos de datos ancho y alto, que son las propiedades que caracterizan a un rectángulo.


El comportamiento de un objeto (también conocido como sus acciones) se define mediante métodos. A Invocar un método en un objeto es pedirle al objeto que realice una acción. Por ejemplo, se puede definir un método llamado getArea() para objetos circulo. Un objeto circulo puede invocar getArea() para devolver su área.


Introduction to java programming, Y. Daniel Liang, Pág. 264


Dado que una clase describe un conjunto de objetos que tienen idénticas características (elementos de datos) y comportamientos (funcionalidad), una clase es realmente un tipo de datos porque un número de punto flotante, por ejemplo, también tiene un conjunto de características y comportamientos. La diferencia es que un programador define una clase para adaptarse a un problema en lugar de ser forzado a usar un tipo de datos existente que fue diseñado para representar una unidad de almacenamiento en una máquina. Se extiende el lenguaje de programación agregando nuevos tipos de datos específicos para sus necesidades. El sistema de programación da la bienvenida a las nuevas clases y les da a todas el cuidado y la verificación de tipos que da a los tipos integrados.


Thinking In C++, Bruce Eckel, Pág. 28-29

Código en java. Se presenta la clase Circulo en el siguiente código:


       public class Circulo {
         double radio;
         double getArea() {
           return radio * radio * Math.PI;
         }
         public static void main(String[] args) {
           // Crea un circulo con radio 1.0
           Circulo circulo1 = new Circulo();
           circulo1.radio = 1;
           System.out.println("El area del circulo de radio "
	     + circulo1.radio + " es " + circulo1.getArea());
           // Crea un circulo con radio 25
           Circulo circulo2 = new Circulo();
           circulo2.radio = 25;
           System.out.println("El area del circulo de radio "
	      + circulo2.radio + " es " + circulo2.getArea());
           // Crea un circulo de radio 125
           Circulo circulo3 = new Circulo();
           circulo3.radio = 125;
           System.out.println("El area del circulo de radio "
	      + circulo3.radio + " es " + circulo3.getArea());
           // Modifica el radio del circulo
           circulo2.radio = 100;
           System.out.println("El area del circulo de radio "
	      + circulo2.radio + " es " + circulo2.getArea() );
         }
       }

Código en java. Se presenta la clase Raton en el siguiente código:


       public class Raton {
         private int edad = 0; // edad del raton en dias
         private double peso = 1.0; // peso del raton en gramos
         private double porcientoCrecimiento; // incremento en dias
  
         // Este metodo asigna el porciento de crecimiento del raton.
         public void setPorcientoCrecimiento(double porcientoCrecimiento) {
           this.porcientoCrecimiento = porcientoCrecimiento; 
         } // fin del setPorcientoCrecimiento
  
        // Este metodo simula un dia de crecimiento del raton.
        public void crecer() {
          this.peso += (.01 * this.porcientoCrecimiento * this.peso); 
          this.edad++;
        } // fin crecer
  
        // Este metodo imprime la edad del raton y su peso.
        public void desplegar() {
          System.out.printf("Edad = %d, peso = %.3f\n",
          this.edad, this.peso);
        } // fin desplegar
      } // fin clase Raton

El objeto

Definición de objeto. En la vida real los objetos son representados (o modelados) como un conjunto de objetos, los cuales forman un modelo de objeto.


Todo es un objeto. Piense en un objeto como una fantasía variable; almacena datos, pero puede "hacer solicitudes" a ese objeto, pidiéndole que realice operaciones sobre sí mismo. En teoria, puede tomar cualquier componente conceptual en el problema que intentas resolver (perros, edificios, servicios, etc.) y representarlo como un objeto en su programa.


Thinking In C++, Bruce Eckel, Pág. 26


Ejemplos de objeto. Un objeto carro puede representar un carro, un objeto cesta puede representar una compra de cestas, y un objeto documento podría representar un documento tal como una página web.


Caractísticas de los objetos. Cada objeto puede tener un conjunto de propiedades que describen aspectos del objeto. Un objeto carro puede tener propiedades que describen su color o el tamaño del motor. Un objeto cesta puede tener propiedades que describen el número de elementos que contiene o el valor de aquellos elementos. Un objeto documento tene propiedades que describen el color de fondo de la página web o el titulo de la página.


Los métodos. Hay métodos; cada método describe una acción que puede ser hecha (o con) el objeto. Por ejemplo, un método de un objeto representa un carro que puede ser acelerado o para cambiar un engranaje. Un método de comprar cesto puede ser agregar un elemento o para recalcular el valor de los elementos en el cesto. Un método de un documento podría ser escribir una linea nueva de texto en la página web.



Los eventos. Finalmente, hay eventos. En un lenguaje de programación, un evento es el objeto que levanta la mano y que dice, "x sucedió", generalmente porque un programa pude hacer algo como un resultado de un evento. Por ejemplo, un objeto carro puede aumentar un evento que dice la ignición ha comenzado o que se acabó la gasolina. Un cesto puede subir un evento cuando un elemento es agregado o cuando esta lleno. Un documento puede aumentar un evento cuando el usuario presiona Submit en forma de click en un enlace. Otras formas, un evento puede también disparar acciones; por ejemplo, si un carro no tiene gasolina, el carro se parará.


Los modelos. Un modelo objeto es por ello una descripción de como un programa puede representar entidades de la vida real que usan un conjunto de objetos, y esto también especifica un conjunto de métodos, propiedades, y eventos que un objeto puede tener.


Los buscadores. Los principales buscadores implementan un modelo objeto llamado DOM que fué ideado para representar páginas web, la página como un todo es representado usando un objeto documento.

Beginnig HTML and CSS, Rob Larsen, Pág. 341

Sin inicialización para la variable de instancia porcientoCrecimiento, ¿cómo puede configurar el crecimiento para un objeto raton? Puede hacer que el objeto raton llame al método setPorcientoCrecimiento con un valor de tasa de crecimiento como argumento. Por ejemplo, así es como un objeto raton puede establecer su tasa de crecimiento en 10 (por ciento):


Código en java. Se presenta la forma de invocar al objeto raton en su método porcientoCrecimiento:


    setPorcientoCrecimiento(10);

Los valores entre paréntesis de una llamada a un método se denominan argumentos. Por tanto, en el ejemplo anterior, 10 es un argumento. El 10 se pasa a la variable porcientoCrecimiento en el encabezado de setPorcientoCrecimiento. Las variables entre paréntesis de un encabezado de método se denominan parámetros. Por lo tanto, en el programa Raton se muestra, porcientoCrecimiento el cuál es un parámetro. Dentro de setPorcientoCrecimiento cuerpo del método (el código entre las llaves de apertura y cierre del método), el parámetro porcientoCrecimiento se asigna a la variable de instancia porcientoCrecimiento. Aquí está la declaración de asignación correspondiente:


Código en java. Se presenta la forma de hacer la declaración y asignación del valor:


     this.porcientoCrecimiento = porcientoCrecimiento;
