La Programación Orientada a Objetos

Definición de paradigma de la Real Academia Española. 2. m. Teoría o conjunto de teorías cuyo núcleo central se acepta sin cuestionar y que suministra la base y modelo para resolver problemas y avanzar en el conocimiento.


Clasificación de la programación. La visión tradicional de un programa de computadora es la de un proceso que ha sido codificado en un formulario que se puede ejecutar en una computadora. Esta visión se originó en el hecho de que las primeras computadoras se desarrollaron principalmente para automatizar un proceso bien definido (es decir, un algoritmo) para el cálculo numérico, y se remonta a las primeras computadoras con programa almacenado. En consecuencia, el proceso de creación de software se vio como una traducción de una descripción de lenguaje "natural" a una secuencia de operaciones que podrían ejecutarse en una computadora. Como muchos dirían que este paradigma sigue siendo la mejor manera de presentar la noción de programación a un principiante, pero a medida que los sistemas se vuelven más complejos, su eficacia en el desarrollo de las soluciones se volvieron sospechosas. Este cambio de perspectiva por parte de los desarrolladores de software sucedió durante un período de tiempo y fue impulsado por varios factores, incluido el alto costo del desarrollo y los esfuerzos constantes para encontrar usos para el software en nuevos dominios. Uno podria argumentar con seguridad que las aplicaciones de software desarrolladas en años posteriores tenían dos caracteristicas:


  1. Comportamiento que fue difícil de caracterizar como un proceso.
  2. Requisitos de confiabilidad, rendimiento y costo que hicieron los desarrolladores originales que no enfrentaron.


Paradigma Orientado a Objetos. Durante el desarrollo de este paradigma, como era de esperar, se probaron y descartaron varias ideas y enfoques. A lo largo de los años, el campo se ha estabilizado para que podamos presentar con seguridad las ideas clave cuya solidez ha resistido el paso del tiempo.


La idea principal. Es el objeto, el cual se agrupa en clases.


Object-Oriented Analysis and Design, Sarnath Ramnath and Brahma Dathan, Pág. 4-5


Cambio de lo estructurado a lo orientado a objetos. Los programas de los años 70 a los 90 crecieron en el número de lineas, debido al incremento del uso de la computadora. Ya que cuando un programa tiene más números de lineas se hace más complejo. Hacer desde el principio un programa grande no es práctico realizarlo, es por eso que se usaron las características que ayudaron a hacer la programación con programas grandes más sencilla. Las características, son: la herencia, la encapsulación y el polimorfismo.


Java manual de referencia, Patrick Naughton y Herbert Schildt, Pág. 6


Los lenguajes que estan bajo el paradigma orientado a objetos. Son: Phyton, Java, C++.


Ejemplo de código de clases con diferentes lenguajes


Código en java. Se presenta la clase Luz en el siguiente código:


       class Luz {
         on();
         off();
         brillantez();
         atenuacion();
       }

Código en java. Se presenta la forma de crear objetos y sus llamadas con código:


       Luz foco;
       foco.on();



Ejemplo de código de objetos


Código en java. Se presenta el código de una clase:


     class Datos {
       int i;
       float f;
       boolean b;
     }

Código en java. Se presenta la forma de crear el objeto S con el siguiente código:


     Datos S = new Datos();

Código en java. Se presenta la forma de llamar al objeto y asignarle valor al objeto S con el siguiente código:


     S.i = 47;
     S.f = 1.1;
     S.b = false;

Piensa en Java, Bruce Eckel, Pág. 62-63


Ejemplo de código de objetos

Código en java. El siguiente código es como se plantean los objetos en la programación orientada a objetos:


         public class Animal {
           string perro;
         }

Se presenta el código que incluye un método:


         public class Animal {
           string perro;
             comer {
               string carne;
             }
         }
